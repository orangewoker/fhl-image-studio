# Component Layer

这里是界面组件实现层，按视觉区域拆分。

## 当前分组

- `layout/`
  - 框架级布局部件，例如头部、底栏、工作区标签栏。

- `panel/`
  - 左侧控制和配置相关组件。

- `canvas/`
  - 画板、工具条、状态栏、源图条。

- `history/`
  - 历史记录和原始响应查看。

- `common/`
  - 通用弹层、toast、上下文菜单等。

## 维护规则

- 组件允许：
  - 从 `state/` 读取状态
  - 从 `platform/` 调宿主能力
  - 从 `lib/` 用纯工具

- 组件不应：
  - 新建平台检测规则
  - 直接实现宿主桥接
  - 在多个目录重复维护同一份平台差异

## 何时新建子目录

满足下面任一情况时考虑新建子目录而不是继续堆文件：

- 同一视觉区块文件数已经明显增多
- 同一组组件共享一套局部约束
- 组件内部出现独立的维护边界，需要单独写 README
