# App Layer

`app/` 是前端的顶层装配层，不是业务实现层。

## 应该放什么

- `App.tsx`
  - 根组件装配。
  - 只负责把 header、workspace、footer、gate 和全局 hooks 组合起来。

- `components/`
  - 只放顶层装配相关的小组件。
  - 例如平台工作区切换、全局拖拽导入覆盖层。

- `gates/`
  - 惰性挂载的模态框、抽屉、一次性入口。
  - 目的是缩小顶层渲染影响面、降低首包压力。

- `hooks/`
  - 全局副作用。
  - 例如 bootstrap、全局快捷键、全局粘贴/拖拽导入、平台视图切换。

## 不应该放什么

- 具体业务 UI。
  - 这些应该回到 `components/` 下的具体分组。

- 宿主桥接实现。
  - 例如 Android invoke、Wails runtime、远程内核请求。
  - 这些应放到 `platform/`。

- 纯工具函数。
  - 放到 `lib/`。

## 判断标准

如果一个文件删除后，应用的“整体装配方式”会变，但某个具体业务区域的实现不会变，那它属于 `app/`。
