# FHL-Image-Studio 方汤圆CLI魔改版 V2.0.0 桌面使用说明

这份文档写给拿到项目文件夹的人，尤其是另一台 Windows 电脑上的 Codex 使用者。目标是：不需要先理解源码，也能启动 UI、配置 FHL、用人工方式生成图片、用 Codex 调 CLI 生成图片。

## 你拿到的是什么

你拿到的是一个独立 Windows 便携源码包：

```text
FHL-Image-Studio方汤圆CLI魔改版V2.0.0\
```

它基于上游开源项目 `Image-Studio1.0.7` 重建，不再继续修旧的 `FHL-Image-Studio方汤圆魔改版1.0.0`。当前版本叫：

```text
FHL-Image-Studio方汤圆CLI魔改版 V2.0.0
```

它有两种使用方式：

1. 人工使用 UI：双击 `一键启动FHL桌面版.cmd`，在浏览器里像普通软件一样填 prompt、点生成。
2. Codex 使用 CLI：让 Codex 执行 `程序文件\image-cli.cmd`，直接从命令行生成图片。

## 运行环境

推荐环境：

- Windows 10 / Windows 11，64 位。
- 能访问 FHL 上游地址：`https://www.fhl.mom`。
- 有可用的 FHL API Key。

这个便携包设计为不安装 EXE、不依赖 Go/Wails。完整交付时建议带上：

- `runtime\node\node.exe`
- `image-studio\frontend\node_modules`
- `程序文件\runtime\cli\gptcodex-image.exe`

正常情况下，对方电脑不需要预装：

- Node.js
- npm
- Go
- Wails
- 桌面 EXE 安装包

如果便携 Node 或 `node_modules` 缺失，`一键启动FHL桌面版.cmd` 会给出错误提示；在有系统 Node/npm 的电脑上，它也可以退回到系统 Node/npm。

## 和原版 Image Studio 的区别

- 默认上游改为 FHL Responses API。
- 默认配置：
  - BASE_URL: `https://www.fhl.mom`
  - API 形态: `responses`
  - 请求策略: `openai`
  - 文本模型: `gpt-5.5`
  - 图像模型: `gpt-image-2`
- 顶栏品牌改为 `FHL-Image-Studio方汤圆CLI魔改版`。
- 顶栏增加 FHL QQ 群 `207550870`、复制群号按钮。
- 顶栏增加“一键配置 FHL API”按钮；未配置时高亮闪烁，配置后停止闪烁。
- 根目录固定有 `input\` 和 `output\`：
  - 输入/参考图保存到 `input\`
  - UI/CLI 输出图保存到 `output\`
  - CLI raw 日志保存到 `output\log\`
- 新增 Codex 可调用的非交互 CLI：`程序文件\image-cli.cmd`。
- 保留上游 1.0.7 的基础比例逻辑：`auto`、`1:1`、`3:2`、`2:3`、`16:9`、`9:16`。
- UI 中间预览图会被高斯模糊，并显示 `服务器信号图像已返回，等待最后结果...`，避免把 partial 误看成最终成品。

如果要把这个文件夹发给别人，建议不要一起发这些私密或临时内容：

```text

.tmp\
.npm-cache\
```

- `程序文件\config\cli.env.local` 可能包含你的 API Key。
- `程序文件\config\fhl-api.local.json` 是本机 UI 私有 API 配置。
- `image-studio\frontend\.local\` 是启动器同步给 Vite 的本机私有配置。
- `output\log\` 可能包含上游 raw 响应和请求细节。
- `.tmp\`、`.npm-cache\`、日志、pid 文件都是本机临时内容。

可以保留这些正式运行文件：

如果不想把历史生成图发给别人，可以清空 `output\`，但保留文件夹本身和 `output\log\` 文件夹结构。

## 一键配置 FHL API

UI 和 CLI 使用两套本机私有配置：

- UI 私有配置：`程序文件\config\fhl-api.local.json`
- CLI 私有配置：`程序文件\config\cli.env.local`

如果是自己电脑，可以提前准备这两个文件；如果发给别人，不要把这两个私有文件一起发出去。

新用户推荐流程：

1. 双击 `一键启动FHL桌面版.cmd`。
2. 打开 `http://127.0.0.1:5173/`。
3. 点击顶栏“一键配置 FHL API”。
4. 粘贴自己的 FHL API Key。
5. 确认 BASE_URL 是 `https://www.fhl.mom`。
6. 确认 API 形态是 `Responses API`。
7. 保存/测试后回到主界面生成图片。

UI 测试或生图跑通后，会自动把完整 CLI 配置写入 `程序文件\config\cli.env.local`，包含 FHL Base URL、Responses API、OpenAI 请求策略、`gpt-5.5 / gpt-image-2`、默认输出参数和用户自己的 API Key。

注意：`cli.env.local` 是本机私有文件，不要发给别人。只有 UI 同步失败时，才把复制 `程序文件\config\cli.env.example` 作为兜底方案。

## 人工 UI 模式怎么用

适合设计师或普通使用者。

1. 双击 `一键启动FHL桌面版.cmd`。
2. 打开 `http://127.0.0.1:5173/`。
3. 如果还没配置 API Key，点击顶栏“一键配置 FHL API”。
4. 回到主界面，选择“文生图”或“图生图”。
5. 在提示词输入框写 prompt。
6. 设置比例、质量、输出格式。
7. 点击“生成”，或按 `Ctrl + Enter`。

生成过程中：

- 如果上游先返回中间预览图，画面会被高斯模糊。
- 模糊图上会显示 `服务器信号图像已返回，等待最后结果...`。
- 最终图返回后，画布会显示清晰成品。

生成成功后：

- 当前图片会显示在中间画布。
- 历史和结果列表会新增记录。
- 原图文件会保存到根目录 `output\`。

## Codex CLI 模式怎么用

适合让 Codex 批量生产图片，或者把这个工具当成本地生图能力。

先进入项目根目录：

```bat
cd /d D:\你的路径\FHL-Image-Studio方汤圆CLI魔改版V2.0.0
```

文生图：

```bat
程序文件\image-cli.cmd --prompt "一个直播卖苹果的男孩" --size 864x1536 --quality medium
```

图生图：

```bat
程序文件\image-cli.cmd --mode edit --image input\ref.png --prompt "改成赛博朋克风格"
```

CLI 成功时 stdout 只输出 JSON，例如：

```json
{"ok":true,"imagePath":"...\\output\\gptcodex-generate-xxx.png","rawPath":"...\\output\\log\\sse-response-xxx.txt","mode":"generate","apiMode":"responses","size":"1024x1024","quality":"medium","outputFormat":"png","sourceEvent":"final","elapsedSec":98.9}
```

Codex 应读取其中的：

```text
imagePath
```

这就是生成好的图片路径。

Codex 还必须把这张图传回对话框预览，格式如下：

```md
![生成图预览](I:/绝对路径/output/gptcodex-generate-xxx.png)
```

CLI 的 `sourceEvent` 可能是：

- `final`：拿到了最终图，推荐作为成品。
- `partial`：只拿到了中间图，CLI 为了不空跑会落盘兜底；可以回传预览，但必须标注为中间图/非最终图，人工 UI 不应把它当成最终成品。

## 文件夹规则

```text
input\        输入图、参考图、图生图原图
output\       所有生成图、导出图、另存图
output\log\   CLI raw 响应日志
config\       配置模板与本机私有配置
runtime\      便携 Node 与 CLI 运行时
```

设计师只需要记住：

- 要给模型看的图，放 `input\`。
- 模型生成出来的图，看 `output\`。

Codex 只需要记住：

- 图生图时用 `--image input\文件名.png`。
- 生图完成后读取 JSON 里的 `imagePath`。
- 生图完成后必须把 `imagePath` 对应图片传回 Codex 对话框预览。

## 常见问题

### 双击 一键启动FHL桌面版.cmd 后一闪而过

用命令行打开项目目录后运行：

```bat
一键启动FHL桌面版.cmd
```

看终端里的错误。如果提示缺少 `runtime\node\node.exe` 或 `node_modules`，说明便携包没有复制完整。

### 端口 5173 被占用

先关闭旧的预览窗口或旧的 `一键启动FHL桌面版.cmd` 终端，再重新启动。

如果仍然占用，可以查看：

```bat
netstat -ano | findstr :5173
```

### UI 能打开，但生成失败

先检查上游配置：

- BASE_URL 是否是 `https://www.fhl.mom`
- API 形态是否是 Responses API
- 请求策略是否是 `openai`
- 文本模型是否是 `gpt-5.5`
- 图像模型是否是 `gpt-image-2`
- API Key 是否填对
- 测试连接是否通过

### CLI 提示 api key must not be empty

说明 CLI 没有读到 API Key。优先回到 UI 同步流程：

```bat
一键启动FHL桌面版.cmd
```

在 UI 里点击“一键配置 FHL API”，填入自己的 API Key，并测试 API 或成功生图一次。UI 跑通后会自动生成 `程序文件\config\cli.env.local`。手动编辑 env 只作为同步失败时的兜底。

### CLI 返回 server_error 或 api_error

这是上游偶发错误。当前 CLI 会对 SSE `server_error` / `api_error` 自动重试。若多次失败，换一个简单 prompt 先测试，例如：

```bat
程序文件\image-cli.cmd --prompt "red apple on white background" --size 1024x1024 --quality medium
```

### Codex 不知道怎么调用

告诉 Codex：

```text
在项目根目录运行 image-cli.cmd，不要用内置 imagegen。成功后读取 stdout JSON 的 imagePath，并用 Markdown 图片语法把生成图传回 Codex 对话框预览。
```

### 换电脑后 UI 有，CLI 没法用

检查：

```text
程序文件\runtime\cli\gptcodex-image.exe
程序文件\config\cli.env.local
```

前者是 CLI 运行文件，后者是 UI 跑通后自动同步出的本机私有 CLI 配置。
