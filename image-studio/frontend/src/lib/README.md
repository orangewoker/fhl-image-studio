# Platform-Agnostic Utilities

`lib/` 只保留平台无关工具。

## 当前允许的内容

- 图像数据辅助
- 本地存储与数据整理
- 配置模型序列化
- 安全清洗与导入导出辅助
- 与 UI/宿主解耦的纯函数

## 不应该再出现的内容

- 平台检测
- Android bridge
- Wails runtime host
- 宿主文件系统 / 原生对话框桥接
- 平台专用 HTTP 通道

这些内容已经迁到 `platform/`。如果未来有人准备在 `lib/` 新增这类文件，说明分层在退化，应直接拒绝。

## 快速判断

如果一个文件需要知道“当前运行在什么宿主/平台”，它通常就不应该留在 `lib/`。
